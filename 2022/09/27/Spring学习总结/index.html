Spring学习总结

發表於2022-09-27|更新於2022-09-27
|閱讀量:

Spring学习总结:

轻量级开源的javaEE框架
两个核心：IOC和Aop
IOC容器：控制反转，创建对象
Aop：面向切面，在不修改源代码下进行功能增强

IOC:降低代码耦合度,把对象创建和对象调用的过程交给spring管理

IOC底层：xml解析、工厂模式、反射

IOC容器：Spring提供IOC容器实现两种实现方式（两个接口）
1.BeanFactory：IOC基本实现，是Spring内部使用的接口
2.ApplicationContext：BeanFactory接口的子接口，提供更多强大的功能

ApplicationContext

IOC操作Bean管理:
两个操作:(1)Spring创建对象
(2)Spring注入属性

两种方式:(1)基于xml配置文件方式体现
(2)基于注解方式体现

基于xml方式创建对象
<bean id=”user” class”com.atguigu.spring5.User”>

1) spring配置文件中，使用bean标签，添加相应属性实现对象创建
2) 属性
   *id属性：唯一标识
   *class属性：类全路径(包和类路径)
   *

基于xml方式注入属性
1）DI：依赖注入，注入属性

创建类(Bean)
定义属性
生成对应set方法

FactoryBean
1.普通bean：配置文件中 bean类型就是返回类型
2.工厂bean：在配置文件定义bean类型可以和返回类型不一样

Bean的作用
1.在spring里，设置创建bean实例是单实例还是多实例

2.scope属性值
singleton，默认值，表示是单实例对象
prototype，表示多实例对象

bean的生命周期
（1）通过构造器创建bean实例
（2）为bean的属性设置值和其他bean引用
（3）调用bean的初始化的方法
（4）bean可以使用（获取到对象）
（5）当容器关闭时候，调用bean的销毁方法(需配置)

Spring针对Bean管理中创建对象提供注解
@Component
@Service
@Controller
@Repository

基于注解方式实现对象创建
第一步 引入spring-aop-5.2.6.RELEASE.jar 依赖
第二步开启组件扫描 xml中引入context:component-scan
第三步创建类

xml
use-default-filters=”false”
<context:include-filter ….>包含哪些需要扫描
<context:exclude-filter ….>不包含的扫描

注解方式注入属性
@AutoWired 根据属性类型进行自动装配
@Qualifier 根据属性名称进行注入
@Resource 可以根据类型注入，也可以根据名称注入
@Value 注入普通类型属性

创建对象new，注入属性set

@AutoWired
@Qualifier和AutoWired一起使用

完全注解开发
1)创建配置类，替代xml文件
@Configuration
@ComponentScan(basePackages = {“com.qs”}
public class SpringConfig
2)编写测试类
ApplicationContext context
= new AnnotationConfigApplicationContext(SpringConfig.class);

AOP(面向切面编程)
使用Proxy类里的方法创建代理对象
1）调用newProxyInstance
第一个参数：类加载器
第二个参数：增强方法所在的类，这个类实现的接口，支持多个接口
第三个参数：实现这个接口InvocationHandler，创建代理对象，写增强方法

编写JDK动态代理代码
1）创建接口，定义方法
2）创建接口实现类，实现方法

Aop术语
1、连接点
类中哪些方法可以被增强
2、切入点
被增强的方法
3、通知(增强)
实际增强的逻辑部分 （日志）
有五种类型：
前置通知
后置通知
环绕通知
异常通知
最终通知
4、切面
把通知应用到切入点的过程

Spring框架一般都是基于AspectJ实现Aop操作
AspectJ不是Spring组成部分，独立Aop框架，一般和Spring一起使用

基于AspectJ实现AOP操作

​ 1)基于xml配置文件实现

​ 2)基于注解方式实现

切入点表达式

​ 语法结构：

​ execution([][][][][权限修饰符][],[返回类型][],[类全路径][],[方法名称][],[参数列表])

​ execution(*com.qs.BookDao.add(..))

​ execution(* com.qs.BookDao.*)

​ execution：执行；实施

操作：

​ 1、创建类，定义方法（ add() ）

​ 2、创建增强类(编写增强逻辑)

​ (1)在增强类中,创建方法，让不同的方法带标不同的数据类型

​ 3、进行通知的配置

​ (1)在spring配置文件中，开启注解扫描

​ (2)使用注解创建User和UserProxy对象

​ (3)在增强类上添加注解@Aspect

​ (4)在spring配置文件中开启生成代理对象

有多个增强类对同一个方法增强，可以设置增强类的优先级

​ 在增强类前添加注解@Order(数字类型值),越小优先级越高
